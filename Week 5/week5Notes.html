Chapter 8 Forms

  • Forms are really useful for information that needs to be gathered from a user. This can be email, name, phone, etc.
  • Forms have inputs, events, and other functions.

  • Some common types of form control are:
    input: fields, including text, passwords, check boxes, radio buttons, and file uploads
    select: menus for drop-down lists of options
    textarea: elements for longer text entry
    button: elements for submitting and resetting forms

  • Forms can also have validation to make sure that all the fields are full or that the input is the right kind of information.

  • Using the disable property we can make sure that a button or field in a form is not clickable until all the actions neccissary are completed.


Chapter 12 Object-Oriented Programming
  • A lot of objects can have functions associated with them to help the object evolve and grow.
  • With constructors you can define the object to have a specific set of parameters and variables. While deconstructors remove all the data that's in that object and resets the object.

  • Inheritance is when an object can take properties and methods from other objects.
    Polymorphism is when different objects can have the same methods and use them in different ways.
    Mixin is adding methods and inheritance to other objects.
